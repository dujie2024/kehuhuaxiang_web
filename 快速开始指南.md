# 快速开始指南 - BS架构Web版本

## 🎯 一分钟快速启动

### 步骤1: 安装依赖包

打开命令提示符（CMD）或PowerShell，进入项目目录：

```bash
cd d:\workspace\kehuhuaxiang_web
pip install -r requirements_web.txt
```

### 步骤2: 启动Web服务器

**方法一：双击批处理文件（最简单）**
```
双击运行: 启动Web服务器.bat
```

**方法二：命令行启动**
```bash
python web_app.py
```

### 步骤3: 打开浏览器访问

启动成功后，会显示类似信息：
```
本地访问地址: http://localhost:5000
局域网访问地址: http://192.168.1.100:5000
```

在Google Chrome浏览器中打开上述地址即可使用！

## 🌐 局域网内其他电脑访问

### 服务器端（运行Web服务的电脑）

1. **启动服务器**（按上述步骤）
2. **记下局域网IP地址**（启动时会显示，如：192.168.1.100）
3. **配置防火墙**（如果无法访问）：
   - 打开"Windows Defender 防火墙"
   - 点击"高级设置"
   - 选择"入站规则" → "新建规则"
   - 选择"端口" → 下一步
   - 选择"TCP"，输入端口号：5000 → 下一步
   - 选择"允许连接" → 下一步
   - 全部勾选 → 下一步
   - 输入名称：新朝旅游Web服务 → 完成

### 客户端（访问的电脑）

1. 确保与服务器在同一局域网（连接同一个路由器/交换机）
2. 打开Google Chrome浏览器
3. 输入地址：`http://服务器IP:5000`
   - 例如：`http://192.168.1.100:5000`
4. 开始使用！

## ❓ 常见问题

### Q1: 无法访问Web页面？

**检查清单：**
- [ ] 服务器是否正常启动？（查看控制台是否显示"Running on..."）
- [ ] 浏览器地址是否正确？
- [ ] 防火墙是否允许端口5000？
- [ ] 客户端和服务器是否在同一局域网？

**解决方法：**
1. 重启服务器
2. 临时关闭防火墙测试
3. 使用`ping 服务器IP`测试网络连通性

### Q2: 查询数据时提示错误？

**可能原因：**
- 数据库连接失败
- 日期范围内没有数据
- 网络连接中断

**解决方法：**
1. 检查`config.ini`数据库配置是否正确
2. 查看`logs`目录下的日志文件
3. 尝试不同的日期范围

### Q3: 图表无法显示？

**解决方法：**
1. 确保先点击"开始分析"获取数据
2. 再点击"生成图表"
3. 检查浏览器控制台是否有错误（按F12打开）

### Q4: 多个用户同时访问会冲突吗？

**不会！** BS架构支持多用户同时访问，每个用户的操作互不影响。

## 🔧 系统要求

### 服务器端
- **操作系统**: Windows 7/10/11, Linux, macOS
- **Python**: 3.7 或更高版本
- **内存**: 建议 2GB 以上
- **网络**: 局域网连接

### 客户端
- **浏览器**: Google Chrome 90+（推荐）
- **其他浏览器**: Firefox 88+, Edge 90+, Safari 14+
- **网络**: 与服务器同一局域网

## 📊 数据范围

- **停车数据**: 2025年1月1日 - 2025年10月10日
- **预订数据**: 2025年1月1日 - 2025年8月20日

## 💡 使用技巧

1. **默认日期**: 系统自动设置为最近30天，可根据需要调整
2. **多次查询**: 可以多次查询不同条件的数据进行对比
3. **图表保存**: 右键点击图表可保存为图片
4. **刷新页面**: 如遇问题，按F5刷新页面重试
