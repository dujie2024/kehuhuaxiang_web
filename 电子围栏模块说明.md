# 电子围栏客流分析模块说明

## 📋 模块概述

电子围栏客流分析模块用于分析景区进出人数的实时数据,帮助了解客流趋势和统计信息。

---

## 🗄️ 多数据库支持

### 配置文件扩展

已将 `config.ini` 扩展为支持多数据库配置:

```ini
[database.carpark]
host = 192.168.0.51
user = root
password = root
database = carpark
port = 3306

[database.dzwl]
host = 192.168.0.51
user = root
password = root
database = dzwl
port = 3306
```

**特点**:
- 使用 `[database.{db_name}]` 格式定义多个数据库
- 每个模块可以独立指定使用哪个数据库
- 数据库与模块非 1对1 关系,灵活配置

### DatabaseModel 改进

```python
class DatabaseModel:
    def __init__(self, config_file='config.ini', db_name='carpark'):
        # 根据 db_name 读取对应的数据库配置
        section_name = f'database.{db_name}'
        # 向后兼容旧配置格式
        if not config.has_section(section_name):
            section_name = 'database'
```

**使用方式**:
```python
# 连接 carpark 数据库
db_carpark = DatabaseModel(db_name='carpark')

# 连接 dzwl 数据库
db_dzwl = DatabaseModel(db_name='dzwl')
```

## 📊 数据查询

### SQL 视图

基于 `joy_dzwl_tab_q2` 表的聚合查询:

```sql
SELECT 
    STR_TO_DATE(`joy_dzwl_tab_q2`.`日期`, '%Y-%m-%d') AS `stat_date`,
    SUM(`joy_dzwl_tab_q2`.`进入累计`) AS `daily_enter_count`,
    SUM(`joy_dzwl_tab_q2`.`离开累计`) AS `daily_exit_count`
FROM `joy_dzwl_tab_q2`
WHERE STR_TO_DATE(`joy_dzwl_tab_q2`.`日期`, '%Y-%m-%d') BETWEEN %s AND %s
GROUP BY STR_TO_DATE(`joy_dzwl_tab_q2`.`日期`, '%Y-%m-%d')
ORDER BY `stat_date`
```

**返回字段**:
- `stat_date`: 统计日期
- `daily_enter_count`: 当日进入人数
- `daily_exit_count`: 当日离开人数

## 🎯 功能特性

### 1. 数据查询
- 按日期范围查询客流数据
- 显示每日进入和离开人数
- 支持快速日期选择(五一/十一/暑假)

### 2. 统计分析
自动计算以下统计指标:
- **统计天数**: 查询范围内的天数
- **总进入人数**: 累计进入人数
- **总离开人数**: 累计离开人数
- **日均进入**: 平均每日进入人数
- **日均离开**: 平均每日离开人数

### 3. 数据展示

**统计卡片**:
```
┌──────────┬──────────┬──────────┬──────────┐
│ 统计天数  │  总进入   │  总离开   │ 日均进入  │
│  30 天   │ 45,000人 │ 43,500人 │ 1,500人  │
└──────────┴──────────┴──────────┴──────────┘
```

**详细列表**:
```
2025-05-01  进入: 2,345 人  离开: 2,234 人
2025-05-02  进入: 3,456 人  离开: 3,321 人
2025-05-03  进入: 4,567 人  离开: 4,432 人
...
```

### 4. 图表展示
- 双折线图展示进入和离开趋势
- 蓝色线: 进入人数
- 橙色线: 离开人数
- 支持悬停查看具体数值

## 🔧 扩展其他数据库

### 添加新数据库

**1. 在 config.ini 中添加配置**:
```ini
[database.new_db]
host = 192.168.0.51
user = root
password = root
database = new_database
port = 3306
```

**2. 在 Model 中使用**:
```python
class NewDataModel:
    def __init__(self):
        # 指定使用 new_db 数据库
        self.db = DatabaseModel(db_name='new_db')
```

**3. 无需修改其他代码**:
- Controller 层调用 Model
- View 层调用 Controller
- 完全解耦,易于扩展
