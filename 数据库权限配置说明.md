# 数据库权限配置说明

## 🔐 安全改进方案

### 问题
当前应用使用 `root` 账户访问数据库，存在安全风险：
- root 权限过大，可以执行任何操作
- 应用只需要查询权限，不需要修改权限
- 如果应用被攻击，数据库完全暴露

### 解决方案
创建专用的应用账户，只授予 SELECT 权限（查询表和视图）。

---

## 📋 实施步骤

### 步骤 1: 执行 SQL 脚本

使用 root 账户连接数据库，执行创建用户的 SQL：

```bash
# 方式一：使用 MySQL 命令行
mysql -u root -p < database_user_setup.sql

# 方式二：登录后执行
mysql -u root -p
source database_user_setup.sql
```

### 步骤 2: 修改应用配置

编辑 `config.ini` 文件，将 root 账户改为新创建的应用账户：

**修改前**:
```ini
[database.carpark]
host = 192.168.0.51
user = root              # ❌ 使用 root 账户
password = root
database = carpark
port = 3306

[database.dzwl]
host = 192.168.0.51
user = root              # ❌ 使用 root 账户
password = root
database = dzwl
port = 3306
```

**修改后**:
```ini
[database.carpark]
host = 192.168.0.51
user = carpark_app       # ✅ 使用应用专用账户
password = carpark_app_2026
database = carpark
port = 3306

[database.dzwl]
host = 192.168.0.51
user = carpark_app       # ✅ 使用应用专用账户
password = carpark_app_2026
database = dzwl
port = 3306
```

### 步骤 3: 测试连接

重启应用，验证数据库连接正常：

```bash
python app.py
```

## 🔑 创建的用户信息

### 用户账户
- **用户名**: `carpark_app`
- **密码**: `carpark_app_2026` (请修改为强密码)
- **主机**: `%` (允许任何主机，建议改为具体IP)

### 授予的权限
- **carpark 数据库**: SELECT 权限（所有表和视图）
- **dzwl 数据库**: SELECT 权限（所有表和视图）

### 权限说明
```sql
-- 可以执行的操作
SELECT * FROM joy_carpark_tab;        -- ✅ 允许
SELECT * FROM joy_booking_tab;        -- ✅ 允许
SELECT * FROM joy_dzwl_tab_q2;        -- ✅ 允许

-- 不能执行的操作
INSERT INTO joy_carpark_tab ...;      -- ❌ 禁止
UPDATE joy_carpark_tab ...;           -- ❌ 禁止
DELETE FROM joy_carpark_tab ...;      -- ❌ 禁止
DROP TABLE joy_carpark_tab;           -- ❌ 禁止
CREATE TABLE ...;                     -- ❌ 禁止
```

## 🛡️ 安全建议

### 1. 修改默认密码

**强密码要求**:
- 长度至少 12 位
- 包含大小写字母
- 包含数字
- 包含特殊字符

**示例**:
```sql
-- 修改密码
ALTER USER 'carpark_app'@'%' IDENTIFIED BY 'Cp@2026#StrongPass!';
FLUSH PRIVILEGES;
```

### 2. 限制访问IP

**推荐配置**:
```sql
-- 删除通配符用户
DROP USER IF EXISTS 'carpark_app'@'%';

-- 创建限制IP的用户
CREATE USER 'carpark_app'@'192.168.0.%' IDENTIFIED BY 'your_password';
GRANT SELECT ON carpark.* TO 'carpark_app'@'192.168.0.%';
GRANT SELECT ON dzwl.* TO 'carpark_app'@'192.168.0.%';
FLUSH PRIVILEGES;
```

### 3. 表级权限（更细粒度）

如果只需要访问特定表：

```sql
-- 只授予特定表的权限
GRANT SELECT ON carpark.joy_carpark_tab TO 'carpark_app'@'%';
GRANT SELECT ON carpark.joy_booking_tab TO 'carpark_app'@'%';
GRANT SELECT ON dzwl.joy_dzwl_tab_q2 TO 'carpark_app'@'%';
FLUSH PRIVILEGES;
```

### 4. 创建独立用户

为不同数据库创建独立用户：

```sql
-- carpark 数据库用户
CREATE USER 'carpark_app'@'%' IDENTIFIED BY 'password1';
GRANT SELECT ON carpark.* TO 'carpark_app'@'%';

-- dzwl 数据库用户
CREATE USER 'dzwl_app'@'%' IDENTIFIED BY 'password2';
GRANT SELECT ON dzwl.* TO 'dzwl_app'@'%';

FLUSH PRIVILEGES;
```

对应的配置文件：
```ini
[database.carpark]
user = carpark_app
password = password1

[database.dzwl]
user = dzwl_app
password = password2
```

## ⚠️ 注意事项

### 1. 应用只读
- 应用只能查询数据，不能修改
- 如果需要写入功能，需要额外授权
- 确保应用逻辑不包含写入操作

### 2. 视图权限
- SELECT 权限包括查询视图
- 视图的底层表也需要有权限
- 确保视图定义者有足够权限

### 3. 存储过程
- 当前配置不包括执行存储过程的权限
- 如需执行存储过程，添加 EXECUTE 权限：
  ```sql
  GRANT EXECUTE ON carpark.* TO 'carpark_app'@'%';
  ```

### 4. 连接数限制
- 可以限制用户的最大连接数：
  ```sql
  ALTER USER 'carpark_app'@'%' WITH MAX_USER_CONNECTIONS 10;
  ```

### 5. 密码策略
- MySQL 8.0+ 默认要求强密码
- 可以配置密码策略：
  ```sql
  -- 查看密码策略
  SHOW VARIABLES LIKE 'validate_password%';
  ```

## 📞 故障排除

### 问题 1: 连接被拒绝
```
ERROR 1045 (28000): Access denied for user 'carpark_app'@'xxx' (using password: YES)
```

**解决**:
- 检查用户名和密码是否正确
- 检查主机是否匹配（% 或具体IP）
- 执行 `FLUSH PRIVILEGES;`

### 问题 2: 权限不足
```
ERROR 1142 (42000): SELECT command denied to user 'carpark_app'@'xxx' for table 'xxx'
```

**解决**:
- 检查是否授予了 SELECT 权限
- 检查数据库名是否正确
- 重新授权并刷新权限

### 问题 3: 无法查询视图
```
ERROR 1356 (HY000): View 'xxx' references invalid table(s) or column(s)
```

**解决**:
- 检查视图底层表的权限
- 确保视图定义者有足够权限
- 使用 DEFINER 子句重建视图
